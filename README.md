# RiskGenie：結合情資之 AI 資安風險精靈

## 一、專題簡介

RiskGenie 是一套支援 ISMS（資訊安全管理系統）運作的 AI 資安風險顧問系統，主要協助組織進行資訊資產盤點、風險評鑑、AI 改善建議產生與風險報表查看。

本系統希望降低非資安背景人員參與資安管理工作的門檻，讓使用者能透過較簡單的操作流程，完成資訊資產管理與風險評估作業。

## 二、專題目標

本專題主要目標如下：

1. 協助使用者建立與維護資訊資產資料。
2. 提供資產盤點檢查功能，提醒資料缺漏。
3. 協助使用者進行風險評鑑。
4. 透過 AI 產生初步改善建議。
5. 提供風險報表與高風險資產查看功能。
6. 支援帳號權限管理與資料備份。

## 三、主要功能

本系統規劃之主要功能包含：

* 使用者登入／登出
* 新增資訊資產
* 刪除資訊資產
* 修改資訊資產
* 查詢資訊資產
* 資產盤點檢查
* 風險評鑑
* AI 改善建議
* 風險報表產生
* 帳號與權限管理
* 系統資料備份

## 四、使用者角色

本系統主要使用者角色包含：

| 角色    | 說明                   |
| ----- | -------------------- |
| 一般使用者 | 維護資訊資產資料、執行資產盤點與風險評鑑 |
| 資安管理員 | 查看風險報表與高風險資產         |
| 高階主管  | 查看整體風險狀況與報表          |
| 系統管理員 | 管理使用者帳號、權限與系統備份      |

## 五、技術架構

本系統預計採用 Web 系統架構進行開發，包含前端介面、後端程式、資料庫與 AI API 串接。

可能使用技術如下：

* 前端：HTML、CSS、JavaScript
* 後端：Python
* 資料庫：SQLite 或 Supabase
* AI 輔助：外部 AI 模型 API
* 版本控制：Git / GitHub

## 六、目前開發狀態

目前專案仍處於學生專題開發階段，主要目標為完成可展示之核心流程，包括資訊資產管理、風險評鑑、AI 改善建議與風險報表等功能。

後續將依照系統分析與設計文件，逐步完成各項功能模組與系統測試。

## 七、備註

本專案為大學畢業專題使用，主要目的為展示 ISMS 風險管理流程與 AI 輔助應用之可行性，並非正式商業化產品。
